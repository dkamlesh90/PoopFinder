Accessibility

Built with a11y in mind

  • 🏷️
    accessibilityLabel on every interactive element
    Screen readers get meaningful context, not just "button".
  • ♿
    accessibilityRole set correctly
    Tabs, buttons, headers, and adjustable swipe cards all declared.
  • 📐
    44pt minimum touch targets
    All tappable elements meet Apple's HIG and Android's Material guidelines.
  • 🔇
    Decorative elements hidden
    Emojis and icon overlays use accessibilityElementsHidden so they don't clutter the reading order.
